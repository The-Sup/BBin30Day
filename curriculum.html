Networking for Hackers — TCP/IP, Ports, Protocols CONCEPTS ▼
// WHAT YOU'RE DOING & WHY

Pentesters attack at the network layer constantly. Understanding TCP handshakes, port states, and protocols isn't theory — it's why your Nmap scans return what they do and how you spot misconfigs.

// TASKS
  • TCP 3-way handshake: SYN → SYN-ACK → ACK (draw it out)
  • Port states: open, closed, filtered — understand what each means for you
  • Memorize critical ports: 21 FTP, 22 SSH, 23 Telnet, 25 SMTP, 53 DNS, 80/443 HTTP/S, 139/445 SMB, 3306 MySQL, 3389 RDP
  • Run Wireshark: capture a ping, a curl to google.com, and analyze the TCP handshake live
  • Understand subnetting: /24 = 254 hosts, /16 = 65534 hosts
// HANDS-ON WIRESHARK TASK
# Capture traffic while you browse, then filter: # In Wireshark display filter bar: tcp.flags.syn == 1 # See all SYN packets http.request.method == "GET" # All HTTP GETs ip.addr == 192.168.56.101 # Filter by target IP

// WHAT YOU'RE DOING & WHY

Random hacking is how amateurs get caught and miss things. Professionals follow a structured kill chain. Understanding PTES and the kill chain gives every action a purpose.

// WHAT YOU'RE DOING & WHY

Nmap is used on every single engagement. You must know it cold — not just the flags but why each scan type behaves differently and what each result tells you.

// ESSENTIAL NMAP COMMANDS
# Host discovery — find live hosts $ nmap -sn 192.168.56.0/24 # Full TCP connect scan (noisy but reliable) $ nmap -sT 192.168.56.101 # SYN stealth scan (requires root, faster) $ nmap -sS 192.168.56.101 # Service version + OS detection + default scripts $ nmap -sV -sC -O 192.168.56.101 # Full port scan with aggressive detection $ nmap -p- -A -T4 192.168.56.101 # UDP scan (slow but critical — DNS/SNMP hide here) $ nmap -sU --top-ports 100 192.168.56.101 # Output to all formats (always save your scans) $ nmap -oA scan_results 192.168.56.101
